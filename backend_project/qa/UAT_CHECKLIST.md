Этот чеклист нужен для ручной проверки перед Demo Day и финальной сдачей.

```bash
docker compose up --build
```

Ожидаемый результат:
- backend доступен на `http://localhost:8000`
- Swagger доступен на `http://localhost:8000/docs`
- frontend доступен на `http://localhost:5173`
- health endpoint возвращает `status=ok`

```bash
export STORAGE_BACKEND=memory

```

```bash
python qa/run_api_contract_tests.py
```

Ожидаемый результат:
- JSON с `passed: 7, failed: 0`

### 2.2 Контекст сотрудника

Статус: ☐ Pass ☐ Fail

## 3. Ручная проверка оценки цели

### 3.1 Слабая цель
Вход:
```json
{
  "employee_id": "emp-001",
  "goal_text": "Улучшить процесс обучения сотрудников",
  "quarter": "Q2",
  "year": 2026
}
```

Ожидание:
- итоговый score ниже, чем у хорошей SMART-цели
- в `recommendations` есть советы по метрике и сроку
- `rewrite` делает цель конкретнее
- `goal_type` не пустой
- `alignment_level` не пустой

Статус: ☐ Pass ☐ Fail

### 3.2 Сильная цель
Вход:
```json
{
  "employee_id": "emp-001",
  "goal_text": "До конца Q2 обеспечить прохождение 95% обязательного обучения сотрудниками производственного блока и снизить долю просроченных обучений ниже 3%",
  "quarter": "Q2",
  "year": 2026
}
```

Ожидание:
- итоговый score выше, чем у слабой цели
- `recommendations` короче или их меньше
- measurable / timebound / relevant не должны быть низкими

Статус: ☐ Pass ☐ Fail

### 3.3 Rewrite endpoint
Вход:
```json
{
  "employee_id": "emp-001",
  "goal_text": "Улучшить адаптацию новых сотрудников",
  "quarter": "Q2"
}
```

Ожидание:
- ответ содержит `rewrite`
- в rewrite есть срок, метрика или более конкретный результат

Статус: ☐ Pass ☐ Fail

---

## 4. Ручная проверка генерации целей

### 4.1 Базовая генерация 3 целей
Вход:
```json
{
  "employee_id": "emp-001",
  "quarter": "Q2",
  "year": 2026,
  "count": 3,
  "focus": "цифровизация и обязательное обучение"
}
```

Ожидание:
- возвращается ровно 3 цели
- у каждой цели есть:
  - `title`
  - `score`
  - `alignment_level`
  - `source_document_title`
  - `source_excerpt`
  - `rationale`
- тексты целей различаются по смыслу
- цели выглядят связанными с контекстом сотрудника

Статус: ☐ Pass ☐ Fail

### 4.2 Проверка grounding
Действия:
- сравнить `source_document_title` и `source_excerpt` с данными после ingestion

Ожидание:
- источник не пустой
- excerpt тематически объясняет, почему цель была предложена
- нет явной галлюцинации по документу

Статус: ☐ Pass ☐ Fail
