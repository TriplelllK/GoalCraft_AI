📌 Проблема

Постановка целей в крупных компаниях — хаотичный процесс:
  • Руководители формулируют цели размыто — «улучшить работу отдела», «повысить эффективность»
  • Нет объективной оценки качества целей — каждый HR судит субъективно
  • Цели не связаны со стратегией компании и ВНД
  • Нет каскадирования — цели руководителя не декомпозируются на подчинённых
  • Дублирование целей между сотрудниками и кварталами
  • Нет аналитики зрелости целеполагания по подразделениям
💡 Масштаб данных хакатона (mock_smart_1.sql): 450 сотрудников, 8 подразделений, 9 000 целей, 160 документов, 30 789 событий

🔍 RAG Pipeline: гибридный поиск по ВНД

160 документов (ВНД) → Sentence chunking
(300 / 50 overlap) → N-gram hashing
(512d vectors) → BM25 index → Hybrid search
Формула: score = cosine × 0.40 + BM25 × 0.35 + keyword_boost × 0.15 + doc_type_bonus × 0.10
КомпонентВесОписание
Cosine similarity0.40N-gram feature hashing (512d)
BM25 score0.35TF × IDF (нормализовано)
Keyword boost0.15Совпадение keywords документа
Doc type bonus0.10strategy > vnd > kpi > policy

Преимущества RAG

  • ✔ Zero-dependency (без BERT/HF)
  • ✔ ~1000x быстрее BERT для русского
  • ✔ Offline-ready (нет внешних API)
  • ✔ Sentence-aware: не режет предложения
  • ✔ Dual-mode: Memory / Qdrant
  • ✔ Поддержка sentence-transformers (опц.)

🎓 Обучение RAG и настройка LLM

RAG — индексация документов

  • Шаг 1: Загрузка ВНД через POST /api/v1/documents/ingest
  • Шаг 2: Sentence-aware chunking (300 chars, overlap 50)
  • Шаг 3: N-gram hash → 512d вектор (SHA-256)
  • Шаг 4: BM25 индекс (token freq + IDF)
  • Данные: 160 документов из mock_smart_1.sql
  • Production: Qdrant + sentence-transformers
Метрики: Recall@5, Precision@5, latency p95

LLM — prompt engineering + fallback

  • Model: GPT-4o-mini (OpenAI)
  • 4 system prompts: Generator, Rewriter, Evaluator, OKR Mapper
  • Graceful degradation: без ключа → rule-based
  • Промпт-контекст: роль + dept + KPI + RAG-чанки
  • Пост-валидация: JSON schema + auto-rewrite < 0.7

Продвинутый режим (опц.)

  • LoRA/SFT на парах «слабая → сильная цель»
  • Обучение на goal_reviews (4 305 рецензий)
  • Корреляция с экспертной разметкой
